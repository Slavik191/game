Написать игру.

Суть игры.
Существует поле в виде сетки настраиваемоего размера.
В некоторых клетках/ячейках сетки расположены склады.
В каждом складе есть определенное кол-во ящиков. Кол-во ящиков отображается внутри клетки.
Так же на поле существует персонаж.
Персонажем можно управлять с помощью стрелок на клавиатуре. Каждая стрелка соответствует направлению движения персонажа на соседнюю клетку.
Если персонаж находится на клетке со складом, то с помощью пробела он может собирать ящики со склада.
Персонаж может выйти из поля. Выход находится снизу от последней клетки на последнем ряду поля. 
После выхода из сетки персонаж может выкладывать ящики на выходе, используя пробел. На выходе показано сколько ящиков персонаж уже перенес.
Персонаж может войти обратно на поле и продолжать собирать ящики.
Игра будет закончена, когда все ящики будут перемещены со складов на выход.

Необходимо использовать ООП для всех элементов игры.
Примерные классы, которые необходимо создать:
Grid - поле. Будет описывать игровое поле, где расположены склады и персонаж.
Character - персонаж. Описывает персонажа, его передвижение, сколько ящиков он сейчас на себе несет.
Store - склад. Описывает склад. Сколько ящиков там хранится.
Game - класс самой игры. Будет отвечать за создание игрового поля, персонажа, складов и за работу со всеми элементами игры. 

Выше описан примерные классы, от которых вы можете отталкиваться. Вы можете использовать такие же классы, а можете придумать свои.

Критерии выполнения задания:
    1. Игра начинается с формы, в которой игрок задает следующие значения: размер поля, кол-во складов, максимальное кол-во ящиков на складе и
    максимальное число ящиков, которые игрок может переносить у себя.
    2. Первоначальное кол-во ящиков на складе задается рандомом, но не должно превышать число, заданное в форме в начале игры.
    Текущее кол-во ящиков на складе отображается в правом верхнем углу ячейки склада.
    Каждый раз, когда персонаж забирает ящик со склада, отображаемое число изменяется и соответствует реальному их кол-ву на складе.
    3. Кол-во ящиков вынесенных с поля отображается на выходе.

Следующие критерии увеличат показатель вашей кармы, но не являются критерями выполнения задания.
Это означает, что если выполнены все следующие критерии, но не выполнены главные критерии, задание не будет считаться выполненным.
    1. Передвижение персонажа анимированно. Тоесть он не телепортируется с клетки на клетку, а плавно перемещается.
    2. С помощью мыши можно перетаскивать ящики со складов прямо на выход. Как это работает - если зажат CTRL, то можно перетащить ящик со склада на выход.
    Для этого необходимо использовать HTML5 Drag And Drop API. Нельзя использовать сторонние библиотеки для работы с drag and drop.

Можно использовать любой CSS фреймворк.
Нельзя использовать React и другие библиотеки или фреймворки.
